# Overview
針對使用者提供的程式碼，補上符合台灣習慣用語的繁體中文註解，並提供簡潔清楚的整體說明。

## Required output
一定要遵守以下原則：
- 使用繁體中文，並採用台灣常見工程溝通用語
- 註解風格清晰、簡潔、中性，不帶情緒
- 不提供 bug 修正、重構、效能、安全性或最佳化建議
- 不改變原始程式邏輯、命名與結構，僅加入必要註解
- 保留原始程式語言與原本縮排風格
- 先提供 markdown 原始碼格式的整體說明，再提供加上註解後的程式碼
- 針對 Python 程式碼加註解時，參考 `references/comment-style-python.md` `assets/sample.py` 的註解風格與密度
- 優先模仿其註解密度、語氣、段落切分與用字
- 只參考風格，不複製範例中的業務語意與命名

## Instructions
1. 先閱讀程式碼，整理出其主要用途、流程、關鍵函式、重要條件分支與資料處理方式
2. 產出一段 markdown 原始碼格式的整體說明，至少包含：
   - `## 程式碼整體說明`
   - `### 主要功能`
   - `### 執行流程`
   - `### 重要補充`
3. 再提供加上註解後的完整程式碼
4. 註解應優先加在 class、function、重要邏輯區塊、條件判斷與關鍵資料處理處
5. 不要對過於直觀的每一行逐行加註解，避免冗長
6. 若程式碼片段不完整，僅針對可理解部分補註解，不延伸猜測未提供邏輯
7. 若程式語言未明確指定，依語法合理判斷並維持該語言常見註解格式

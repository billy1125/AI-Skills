# Output Formats for Polishing and Diagnosis

Use this file to produce consistent responses for academic English polishing and paragraph diagnosis.

## Default polishing output

When the user asks for polishing only, provide:

1. Revised version
2. Brief notes, only if useful

Template:

## Revised version

[Revised paragraph]

## Notes

- [One concise note about tone, logic, or major revision]
- [Optional second note]

If the user asks for only the revised text, provide only the revised text.

## Paragraph diagnosis output

When the user asks for diagnosis, provide:

## Overall assessment

[One to three sentences assessing focus, logic, evidence, and tone.]

## Main issues

1. [Issue]
2. [Issue]
3. [Issue]

## Sentence-level comments

- Sentence 1: [Comment]
- Sentence 2: [Comment]
- Sentence 3: [Comment]

## Revision strategy

[Concrete guidance on how to improve the paragraph.]

## Possible revision

[Revised paragraph, if useful.]

## Combined diagnosis and polishing output

When the user asks for both diagnosis and revision, provide diagnosis first, then revised version.

Template:

## Diagnosis

[Concise diagnosis]

## Revised version

[Revised paragraph]

## What changed

- [Change 1]
- [Change 2]
- [Change 3]

## Edit intensity labels

Use these labels when explaining revision scope:

- Light polish: grammar, fluency, minor word choice.
- Moderate polish: sentence structure, transitions, academic tone.
- Substantive revision: paragraph logic, argument order, evidence interpretation.

## Tone requirements

Feedback should be:
- neutral
- specific
- non-dismissive
- evidence-based
- limited to the text provided

Avoid:
- praising excessively
- speculating about the author's intent beyond the text
- adding new literature or claims without request

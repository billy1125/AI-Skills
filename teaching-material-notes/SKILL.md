# 教材筆記

## 用途

將使用者提供的教材轉換為清晰、結構化的學習筆記。支援的輸入包括上傳的 Word、PowerPoint、PDF 檔案和貼上的網頁 URL。支援的輸出包括摘要、講義大概要、章節式筆記和重點整理。

## 來源規則

1. 僅使用使用者明確上傳的檔案和 URL。

2. 請勿使用 Google 雲端硬碟、YouTube、電子郵件、Slack 或其他連接器。

3. 如果使用者提供 URL，請在進行摘要之前瀏覽或開啟該頁面。當環境要求引用時，請註明網路資源。

4. 如果使用者上傳 PDF 文件，且視覺佈局（例如表格、圖表或類似幻燈片的頁面）至關重要，請在可用時檢查渲染後的頁面或螢幕截圖，而不僅依賴提取的文字。

5. 如果使用者上傳 Word 或 PowerPoint 文件，請擷取並使用文件或投影片的實際內容。盡可能保留標題、投影片標題、編號結構、講者備註和說明文字。

6. 如果提供了多個文件或 URL，請先建立一個整合的來源圖：來源標題、內容類型、主題以及它與其他資料的關係。

## 輸出選擇

1. 先詢問使用者他們想要什麼類型的輸出：摘要、講義大概要、章節式筆記或重點整理。

2. 如果使用者未明確指定，則詢問他們想要的是摘要、講義大綱、章節式筆記還是重點整理。

3. 再詢問要甚麼樣的檔案格式：純文字、Markdown、HTML、Word、PowerPoint。

4. 如果使用者指定了一種或多種輸出類型，請僅產生這些類型。

5. 如果使用者要求的是廣泛的「整理教材」並希望立即得到結果，則預設產生摘要加上重點整理。

6. 除非使用者單獨提出超出此技能預設範圍的要求，否則不要產生複習題、測驗、記憶卡、答案、練習或考試題型。

## 工作流程

1. 識別所有資源。

- 列出每個上傳的 Word/PPT/PDF 檔案以及每個提供的 URL。

- 清楚記錄缺失或無法存取的資源。

2. 提取並規範化內容。

- 保留有意義的章節、標題和投影片的原始順序。

- 刪除重複的頁首、頁尾、導覽文字、版權聲明和無關的網站邊框。

- 保留定義、公式、步驟說明、框架、範例和警告。

3. 建構教學內容圖。 4. 產生所需輸出。

- 確定主題、學習目標、預備知識和主要章節。

- 合併不同來源中重疊的章節。

- 標記矛盾之處、過時的章節或來源缺失，而不是默默地解決它們。

5. 產生所需輸出。

- 當使用者使用中文輸入時，預設使用繁體中文。

- 保持中立的指導性語氣。

- 使用簡潔的標題和完整的句子。

- 除非使用者要求補充，否則避免添加外部知識；任何補充內容都應明確標記為外部內容。

6. 最終回覆前進行品質檢查。

- 確認所有主要來源章節均已包含。

- 確保輸出不包含未經證實的論點。

- 確保預設不包含「可重複問答」、「測驗」、「閃卡」或「考試題」等內容。

## 輸出模板

使用 `references/output_templates.md` 取得四種支援的格式。根據實際來源結構調整模板，但要保持輸出類型清晰可見。

## 處理多個來源

整合多個文件或 URL 時，除非另有要求，否則不要單獨總結每個文件。將它們合併成一個連貫的教學結構：

- 將共同的概念放在一起。

- 以最完整的來源作為框架。

- 在相關章節下方加入其他來源的佐證範例或細節。

- 僅當源之間的差異影響解釋或教學順序時才提及。

## 處理薄弱或不完整的來源

如果擷取不完整，請準確說明缺少的內容：例如無法閱讀的頁面、僅包含圖像的幻燈片、無法存取的 URL、缺少演講者備註或不支援的文件結構。繼續處理可用內容，並將結果標記為「部分」。

## 格式標準

- 建議為中國使用者使用繁體中文。

- 使用中性的學術用語。

- 保持結構清晰：在適當的地方使用標題、副標題和項目符號。

- 不要過度壓縮技術概念。

- 確保範例與來源材料緊密相關。 - 請勿捏造引用、頁碼、投影片編號或章節標題。
